How It Works: Step-by-Step

User Interface

Users initiate interaction through a sleek and intuitive chat-style interface. They can type in everyday language, for example:

  • "What is Article 21?"
  • "Can I be arrested without a warrant?"

The frontend is designed for clarity and ease of use, eliminating legal jargon and encouraging users—regardless of legal knowledge—to explore their rights and obligations under Indian law..

Prompt Sent to Gemini

Once a user submits a query, the input is programmatically formatted into a well-structured prompt and sent to the Gemini API (e.g., gemini-1.5-pro or gemini-2.0-flash). This step involves:

  • Minimal prompt engineering to maintain clarity
  • Adding brief instructions like "Act as an Indian law assistant..." to ensure relevance
  • Filtering unsafe content

This ensures the model receives a clean, context-driven request, improving the quality of the output.

Dataset Lookup

Before relying on generative AI, the system performs a local dataset lookup to find direct matches for frequently asked legal topics. This includes:

  • IPC sections
  • Constitutional references
  • Trusted public FAQs

If a high-confidence match is found, it is used as reference content. This is part of the Retrieval phase in the "RAG (Retrieval-Augmented Generation)" architecture.

RAG + Gemini

If no exact dataset match is found, the system seamlessly invokes the Gemini LLM. Using the RAG approach, the system:

  • Retrieves semantically related text chunks from the dataset (even partial matches)
  • Feeds those into Gemini as context
  • Gemini then generates a response grounded in the legal context

allows the model to provide context-aware, India-specific legal answers that are both factual and aligned with constitutional and criminal law principles.

Final Output

The final response—either generated via Gemini or fetched from the dataset—is:

  • Displayed to the user in real time
  • Rendered in the same chat interface for a smooth conversational flow
  • Designed to be legally accurate, concise, and understandable

This creates an experience where users feel like they're consulting a virtual legal expert, while the system silently handles data retrieval, LLM invocation, and filtering in the background.
